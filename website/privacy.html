Privacy Policy

Last updated: May 25, 2025  ·  VelocityRL by bitsfdb

VelocityRL does not collect personal information. No accounts are required, no tracking pixels are used, and no advertising data is gathered. This policy explains specifically what minimal data does leave your machine and why.

1. The Desktop App

VelocityRL is a local application — all item swapping happens entirely on your machine. The app communicates with the internet in two ways:

1a. Item catalog fetch

On first launch (and periodically thereafter) the app makes a single GET request to api.velocityrl.tech/items.json to download the Rocket League item database. This is a standard HTTPS request. The response is cached locally in your app data folder; the app works offline once cached. No data about you is sent in this request beyond what a normal HTTP request includes (your IP address and the request timestamp, which are logged by our server for up to 7 days as described in section 3).

1b. Crash and error diagnostics

When specific errors occur, the app sends a small diagnostic report to our API. Diagnostics are only sent on failure — never during normal use. Each report contains:

  • The event type (swap_fail, init_fail, or integrity_fail)
  • App version, operating system, and CPU architecture
  • A Unix timestamp
  • On swap failure: the item IDs involved (numeric identifiers, not item names), the folder name of your game directory (not the full path), the engine exit code, and engine stderr/stdout output
  • On init failure: the error message and stack trace from the app runtime
  • On integrity failure: the expected and actual SHA-256 checksums of the engine binary

Diagnostic reports do not contain your name, email, Epic account, file system paths beyond the game folder name, or any other personally identifying information.

1c. Epic catalog fetch (optional)

The app has an optional feature to fetch your owned Rocket League items directly from Epic Games. If you use this feature, you enter an Epic auth token manually. That token is passed only to the local velocity-engine sidecar process, which communicates directly with Epic's servers. The token is never sent to VelocityRL's servers and is not stored on disk.

1d. Local storage

The app saves one configuration file to your local app data directory containing only the path to your Rocket League game folder. This file never leaves your machine.

2. The Website (velocityrl.tech)

The website is a static HTML/CSS page served directly from our server. It loads fonts from Google Fonts (fonts.googleapis.com), which means Google receives a request including your IP address and browser user-agent when the page loads. If you want to avoid this, you can block the font request — it has no effect on functionality.

The website does not use cookies, tracking scripts, analytics platforms (no Google Analytics, no Plausible, no similar services), or advertising networks.

3. The Public API (api.velocityrl.tech)

The VelocityRL API is a public, unauthenticated REST API serving Rocket League item data. When you make a request to the API:

  • Your IP address is recorded in memory to enforce the rate limit (60 requests per minute). This in-memory record is automatically purged after the window expires and is not written to disk.
  • Standard web server access logs are kept for up to 7 days. These logs contain the request path, HTTP status code, response size, timestamp, IP address, and user-agent string. They are used only for debugging and are not shared with third parties.

The API has no authentication, no user accounts, and no persistent record of who requested what.

4. Data sharing

We do not sell, rent, or share any data with third parties. Diagnostic data and server logs are used exclusively for debugging VelocityRL and are accessible only to the project maintainer.

5. Data retention

  • Server access logs: deleted after 7 days
  • Rate-limit records: purged from memory after 1 minute
  • Diagnostic reports: retained indefinitely for debugging; contain no personal data

6. Children

VelocityRL does not knowingly collect any information from users under the age of 13. If you believe a minor has submitted diagnostic data and would like it removed, contact us at the address below.

7. Changes to this policy

If this policy changes in a meaningful way, we will update the date at the top and note the change in the GitHub repository. Continued use of the app or API after a change constitutes acceptance of the updated policy.
